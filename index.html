Refatoração do sistema Olhoneles

Download as .zip Download as .tar.gz View on GitHub

Proposta do projeto

  • Criação de uma Stylesheet baseada na documentação oficial do Python (acessível através do link: https://www.python.org/dev/peps/pep-0008/);
  • Aplicação da Stylesheet que levará a refatorações em termos de:
    • nome de variáveis, atributos, classes, métodos, pacotes, módulos, exceptions, constantes,
    • espaçamentos;
    • linhas em branco;
    • espaços em branco;
    • comentários (em bloco e em linha);
    • e demais recomendações da linguagem. *Encapsulamento de códigos repetitivos em funções atômicas;
  • Remover números mágicos utilizando constantes;
  • Utilizar padrões de projeto (se identificáveis) de modo a tornar as intenções do código mais explícitas;
  • Utilizar padrões da linguagem para melhorar a qualidade do código.
  • Código simples com boa apresentação;
  • Escolher nomes significativos;
  • Decomposição em funções atômicas;
  • Tipos descritivos;
  • Ênfase para código importante;
  • Agrupamento de informações relacionadas;
  • Utilização de Cabeçalho de Arquivo;
  • Tratamento de erros apropriado;
  • Utilização de comentários efetivos.

Durante o desenvolvimento deste trabalho, serão buscados sinais para refatoração, são eles:

  • violações de princípios de boa programação;
  • DRY (Don’t Repeat Yourself);
  • SRP (Single Responsibility Principle);
  • OCP (Open Closed Principle): Um código deve ser aberto para extensão e fechado para alteração. Alterações podem causar efeitos colaterais;
  • KISS (Keep it Short and Simple);
